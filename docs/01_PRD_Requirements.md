# MagpieAuth - 产品需求文档 (PRD)

## 1. 产品简介
MagpieAuth 是一款极致轻量的本地化密码与双重身份验证 (2FA) 管理软件。旨在为用户提供一个纯本地、不依赖云端的高安全性凭据保险箱。通过结合系统级的 PIN 码验证以及极简优雅的用户界面，MagpieAuth 让用户能够安全、高效地管理敏感信息。

## 2. 核心原则
- **绝对的本地化 (Local-First)**：所有数据必须且只能存在于本地磁盘，任何情况下绝不主动发起不受用户感知的外部网络请求上传数据。
- **极致的轻量化 (Lightweight)**：安装包小，内存占用低，启动迅速，不捆绑臃肿的多余功能。
- **无感的高安全 (Seamless Security)**：依托操作系统的内置安全机制（如凭据管理器）进行身份验证，不自己发明加密验证轮子。

## 3. 功能需求详细说明

### 3.1 密码管理
- **条目字段**：
  - **名称 (Name)**：用户自定义的条目标签/名字（如"Google 个人账号"）。
  - **账号 (Account)**：用户名、邮箱或手机号等登录凭证。
  - **密码 (Password)**：对应的登录密码，支持隐藏/显示切换，支持一键复制。
  - **创建时间 (Created At)**：系统自动记录的条目创建时间（也可支持修改时间记录）。
- **条目操作**：
  - 支持新增、查看、编辑、删除密码条目。
  - 支持通过名称或账号快速检索。

### 3.2 2FA (TOTP) 管理
- **标准支持**：完全兼容基于时间的一次性密码算法 (TOTP, RFC 6238)。
- **添加方式**：
  - **扫描二维码**：支持从屏幕截取/选择图片扫描二维码自动解析 `otpauth://` 链接。
  - **手动输入**：支持手动输入密钥 (Secret Key) 及对应的发行方 (Issuer) 和账号名称。
- **展示与使用**：动态显示当前 6 位验证码，提供可视化的倒计时进度条，支持一键复制验证码。

### 3.3 主密码 (PIN 码) 机制与系统集成
- **访问控制**：应用启动或在后台闲置一段时间后，必须通过验证才能访问密码库。
- **验证方式（关键）**：
  - 本软件**不**独立管理主密码的哈希校验。
  - 在获取/显示核心密码明文前，**必须直接调用操作系统的凭据管理器 (OS Credential Manager / Windows Hello / 系统的 PIN 码验证接口)** 来验证当前用户的身份。
  - **规则**：如果主密码/系统 PIN 码输入错误或拒绝验证，软件将锁定界面并拒绝解密或获取核心数据。

### 3.4 界面设计 (UI/UX)
- **极简主义 (Minimalist)**：去除一切不必要的视觉干扰，专注于数据本身。
- **高级质感 (Premium UI)**：拟态、毛玻璃效果、流畅的微交互动画。
- **主题支持**：原生支持并默认适配 **暗黑模式 (Dark Mode)**，完美契合夜间或专业开发环境使用。
- **配色基调**：经典极简的黑/白/灰/银搭配磨砂玻璃 (Frosted Glass) 质感。

### 3.5 数据的导入与导出
- **导出机制**：
  - 用户可随时将整个密码库与 2FA 数据导出为单文件备份。
  - **加密标准**：导出的文件必须使用 **AES-256-GCM** 算法进行全量加密。
  - **导出验证**：导出时不使用系统 PIN 码，而是**要求用户在导出当下，临时设定一个强密码**。该密码将通过密钥派生函数（如 Argon2id 或 PBKDF2）生成 AES 加密密钥。
- **导入机制**：
  - 用户导入备份文件时，需输入当时的导出密码解密数据。
  - 解密成功后，数据将重新并入当前的本地密码库，后续访问继续受系统 PIN 码/凭据管理器的保护。

## 4. 非功能需求
- **运行环境**：优先且主要适配 Windows 平台（基于用户 OS 设定），可预留跨平台（macOS/Linux）的架构兼容性。
- **数据持久化**：使用本地轻量级数据库（如 SQLite）或加密 JSON 存储结构化数据。
- **离线可用**：完全无需联网即可完成 100% 的功能操作。
